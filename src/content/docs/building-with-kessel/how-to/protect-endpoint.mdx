Protect your API endpoints by checking user permissions before granting access. You'll define permissions in KSL, initialize the SDK, and add permission checks to your middleware.

## Prerequisites

- Basic understanding of the [Kessel Schema Language (KSL)](/docs/building-with-kessel/concepts/schema) for defining resources with permissions

## Define your service permissions

The base RBAC schema (`rbac.ksl`) is already deployed to Kessel, so you only need to register your service's permissions.

### Register workspace-level permissions

Create a schema file for your service. For example, `notifications.ksl`:

```ksl
version 0.1
namespace notifications

import rbac

@rbac.add_v1_based_permission(app:'notifications', resource:'integrations', verb:'read', v2_perm:'notifications_integrations_view');
@rbac.add_v1_based_permission(app:'notifications', resource:'integrations', verb:'write', v2_perm:'notifications_integrations_edit');
```

In this schema:
- **`import rbac`** brings in the existing RBAC workspace hierarchy
- **`app`** is your service name (`notifications`)
- **`resource`** is the resource type within your service (`integrations`)
- **`verb`** is the action—typically `read` or `write`
- **`v2_perm`** is the permission name for Check API calls, following the pattern `{app}_{resource}_{view|edit}`

These permissions can be checked at the workspace level and are automatically inherited through the workspace hierarchy.

<Aside type="note">
**Schema deployment** is handled separately from application code. The schema definition process is evolving—for current examples, see the [base schemas in rbac-config](https://github.com/RedHatInsights/rbac-config/tree/master/configs/prod/schemas/src) ([kessel.ksl](https://github.com/RedHatInsights/rbac-config/blob/master/configs/prod/schemas/src/kessel.ksl), [rbac.ksl](https://github.com/RedHatInsights/rbac-config/blob/master/configs/prod/schemas/src/rbac.ksl)).
</Aside>

<Aside type="tip">
To understand how permissions are evaluated through the relationship graph, see [Relationships and permissions](/docs/building-with-kessel/concepts/relationships-permissions/).
</Aside>

## Implement permission checks

Use `Check()` for reads and `CheckForUpdate()` for writes. The difference is consistency: reads can tolerate slight staleness, writes need the latest state.

### When to use Check vs CheckForUpdate

| Method | Consistency | Use For | Example Operations |
|--------|-------------|---------|-------------------|
| **Check()** | Eventually consistent (configurable) | Read operations, UI visibility decisions, standard authorization | GET requests, list filtering, showing/hiding UI elements |
| **CheckForUpdate()** | Strongly consistent (always) | Write operations, sensitive operations | PUT/POST/DELETE requests, resource modification, credential access |

<Aside type="tip">
For a deeper understanding of consistency guarantees, replication windows, and advanced consistency modes, see the [Consistency model](/docs/building-with-kessel/concepts/consistency/) guide.
</Aside>

### Check: Read operations

Use `Check()` for reads. Eventual consistency is fine here—users won't notice a few hundred milliseconds of staleness.

<Tabs syncKey="language">
<TabItem label="Go">

```go
import (
	"context"
	"log"

	"github.com/project-kessel/kessel-sdk-go/kessel/inventory/v1beta2"
	v2 "github.com/project-kessel/kessel-sdk-go/kessel/rbac/v2"
)

func checkReadPermission(ctx context.Context, client v1beta2.KesselInventoryServiceClient, userID, workspaceID string) (bool, error) {
	checkRequest := &v1beta2.CheckRequest{
		Object:   v2.WorkspaceResource(workspaceID),
		Relation: "notifications_integrations_view",
		Subject:  v2.PrincipalSubject(userID, "redhat"),
	}

	response, err := client.Check(ctx, checkRequest)
	if err != nil {
		return false, err
	}

	return response.Allowed == v1beta2.Allowed_ALLOWED_TRUE, nil
}
```

**HTTP middleware**

```go
package main

import (
	"context"
	"net/http"
	"os"

	"github.com/project-kessel/kessel-sdk-go/kessel/inventory/v1beta2"
	v2 "github.com/project-kessel/kessel-sdk-go/kessel/rbac/v2"
)

var kesselClient v1beta2.KesselInventoryServiceClient

func init() {
	client, conn, err := v1beta2.NewClientBuilder(os.Getenv("KESSEL_ENDPOINT")).
		Insecure().
		Build()
	if err != nil {
		panic(err)
	}
	kesselClient = client
	// Note: In production, handle conn.Close() properly
}

// RequirePermission middleware checks if user has required permission
func RequirePermission(relation string, next http.HandlerFunc) http.HandlerFunc {
	return func(w http.ResponseWriter, r *http.Request) {
		// Extract user from header (from your auth middleware)
		userID := r.Header.Get("X-User-ID")
		if userID == "" {
			http.Error(w, "Unauthorized", http.StatusUnauthorized)
			return
		}

		// Extract workspace ID from header or request context
		workspaceID := r.Header.Get("X-Workspace-ID")
		if workspaceID == "" {
			http.Error(w, "Missing workspace", http.StatusBadRequest)
			return
		}
		
		// Check permission
		checkReq := &v1beta2.CheckRequest{
			Object:   v2.WorkspaceResource(workspaceID),
			Relation: relation,
			Subject:  v2.PrincipalSubject(userID, "redhat"),
		}
		
		response, err := kesselClient.Check(context.Background(), checkReq)
		if err != nil {
			http.Error(w, "Permission check failed", http.StatusInternalServerError)
			return
		}
		
		if response.Allowed != v1beta2.Allowed_ALLOWED_TRUE {
			http.Error(w, "Forbidden", http.StatusForbidden)
			return
		}
		
		// Permission granted
		next(w, r)
	}
}

func main() {
	http.HandleFunc("GET /integrations/{id}", RequirePermission("notifications_integrations_view", getIntegrationHandler))
	http.HandleFunc("PUT /integrations/{id}", RequirePermission("notifications_integrations_edit", updateIntegrationHandler))
	
	http.ListenAndServe(":8080", nil)
}

func getIntegrationHandler(w http.ResponseWriter, r *http.Request) {
	// Handle GET request
}

func updateIntegrationHandler(w http.ResponseWriter, r *http.Request) {
	// Handle PUT request
}
```

**Flask decorator**

```python
from flask import Flask, request, abort
from functools import wraps
from kessel.inventory.v1beta2 import (
    ClientBuilder,
    check_request_pb2,
    allowed_pb2,
)
from kessel.rbac.v2 import principal_subject, workspace_resource
from connectrpc.errors import ConnectError

app = Flask(__name__)

# Initialize Kessel client
stub, channel = ClientBuilder("localhost:9000").insecure().build()

def require_permission(relation: str):
    """Decorator to require permission check before executing endpoint."""
    def decorator(f):
        @wraps(f)
        def decorated_function(*args, **kwargs):
            # Extract user from request headers
            user_id = request.headers.get("X-User-ID")
            user_domain = request.headers.get("X-User-Domain", "redhat")
            
            # Extract workspace from header
            workspace_id = request.headers.get("X-Workspace-ID")
            
            if not user_id or not workspace_id:
                abort(400, "Missing required headers")
            
            try:
                # Build check request
                check_req = check_request_pb2.CheckRequest(
                    subject=principal_subject(user_id, user_domain),
                    relation=relation,
                    object=workspace_resource(workspace_id),
                )
                
                # Perform permission check
                response = stub.Check(check_req)
                
                if response.allowed != allowed_pb2.ALLOWED_TRUE:
                    abort(403, f"Permission denied for relation '{relation}'")
                
                # Permission granted, proceed
                return f(*args, **kwargs)
                
            except ConnectError as e:
                abort(500, f"Permission check failed: {e.message}")
        
        return decorated_function
    return decorator

@app.route("/integrations/<integration_id>", methods=["GET"])
@require_permission(relation="notifications_integrations_view")
def get_integration(integration_id):
    return {"integration_id": integration_id, "name": "Example Integration"}

@app.route("/integrations/<integration_id>", methods=["PUT"])
@require_permission(relation="notifications_integrations_edit")
def update_integration(integration_id):
    return {"status": "updated"}

with channel:
    app.run(debug=True)
```

from fastapi import FastAPI, Depends, HTTPException, Header

        raise HTTPException(status_code=500, detail=f"Permission check failed: {e.message}")

**Express middleware**

```javascript
import express from "express";
import { ClientBuilder } from "@project-kessel/kessel-sdk/kessel/inventory/v1beta2";
import { principalSubject, workspaceResource } from "@project-kessel/kessel-sdk/kessel/rbac/v2";

const app = express();

// Initialize Kessel client
const client = new ClientBuilder(process.env.KESSEL_ENDPOINT)
  .insecure()
  .buildAsync();

// Middleware to require permission check
function requirePermission(relation) {
  return async (req, res, next) => {
    // Extract user from header (from your auth middleware)
    const userId = req.headers["x-user-id"];
    const userDomain = req.headers["x-user-domain"] || "redhat";
    
    // Extract workspace from header
    const workspaceId = req.headers["x-workspace-id"];
    
    if (!userId || !workspaceId) {
      return res.status(400).json({ error: "Missing required headers" });
    }
    
    try {
      // Build check request
      const checkRequest = {
        subject: principalSubject(userId, userDomain),
        relation: relation,
        object: workspaceResource(workspaceId),
      };
      
      // Perform permission check
      const response = await client.check(checkRequest);
      
      if (response.allowed !== "ALLOWED_TRUE") {
        return res.status(403).json({ error: `Permission denied for relation '${relation}'` });
      }
      
      // Permission granted, proceed
      next();
    } catch (error) {
      console.error("Permission check failed:", error);
      return res.status(500).json({ error: "Permission check failed" });
    }
  };
}

app.get("/integrations/:id", 
  requirePermission("notifications_integrations_view"),
  (req, res) => {
    res.json({ integration_id: req.params.id, name: "Example Integration" });
  }
);

app.put("/integrations/:id",
  requirePermission("notifications_integrations_edit"),
  (req, res) => {
    res.json({ status: "updated" });
  }
);

app.listen(3000, () => console.log("Server running on port 3000"));
```

```java
import org.springframework.stereotype.Component;
import org.springframework.web.filter.OncePerRequestFilter;
import jakarta.servlet.FilterChain;
import jakarta.servlet.ServletException;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import org.project_kessel.api.inventory.v1beta2.*;
import org.project_kessel.api.rbac.v2.Utils;
import static org.project_kessel.api.inventory.v1beta2.KesselInventoryServiceGrpc.KesselInventoryServiceBlockingStub;
import java.io.IOException;

@Component
public class PermissionFilter extends OncePerRequestFilter {
    
    private final KesselInventoryServiceBlockingStub kesselClient;
    
    public PermissionFilter(KesselInventoryServiceBlockingStub kesselClient) {
        this.kesselClient = kesselClient;
    }
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                   HttpServletResponse response, 
                                   FilterChain filterChain) 
                                   throws ServletException, IOException {
        
        String userId = request.getHeader("X-User-ID");
        String userDomain = request.getHeader("X-User-Domain");
        if (userDomain == null) userDomain = "redhat";
        String workspaceId = request.getHeader("X-Workspace-ID");
        
        if (userId == null || workspaceId == null) {
            response.setStatus(HttpServletResponse.SC_BAD_REQUEST);
            response.getWriter().write("{\"error\":\"Missing required headers\"}");
            return;
        }
        
        // Determine required relation based on endpoint and method
        String relation = getRequiredRelation(request);
        if (relation == null) {
            filterChain.doFilter(request, response);
            return;
        }
        
        try {
            CheckRequest checkRequest = CheckRequest.newBuilder()
                .setSubject(Utils.principalSubject(userId, userDomain))
                .setRelation(relation)
                .setObject(Utils.workspaceResource(workspaceId))
                .build();
            
            CheckResponse checkResponse = kesselClient.check(checkRequest);
            
            if (checkResponse.getAllowed() != Allowed.ALLOWED_TRUE) {
                response.setStatus(HttpServletResponse.SC_FORBIDDEN);
                response.getWriter().write("{\"error\":\"Permission denied\"}");
                return;
            }
            
            filterChain.doFilter(request, response);
            
        } catch (Exception e) {
            response.setStatus(HttpServletResponse.SC_INTERNAL_SERVER_ERROR);
            response.getWriter().write("{\"error\":\"Permission check failed\"}");
        }
    }
    
    private String getRequiredRelation(HttpServletRequest request) {
        String path = request.getRequestURI();
        String method = request.getMethod();
        
        if (path.startsWith("/integrations/")) {
            if ("GET".equals(method)) {
                return "notifications_integrations_view";
            } else if ("PUT".equals(method) || "POST".equals(method) || "DELETE".equals(method)) {
                return "notifications_integrations_edit";
            }
        }
        
        return null; // No permission required
    }
}
```

</TabItem>
</Tabs>

<Aside type="caution">
These examples use `Check()` for simplicity. In production, call `CheckForUpdate()` for writes (PUT/POST/DELETE) instead.
</Aside>

## Handle errors

If Kessel is unavailable, decide whether to fail closed (deny access) or fail open (allow access). Fail closed is safer for most cases.

<Tabs syncKey="language">
<TabItem label="Go">

```go
func checkPermissionSafe(ctx context.Context, client v1beta2.KesselInventoryServiceClient, req *v1beta2.CheckRequest) bool {
	response, err := client.Check(ctx, req)
	if err != nil {
		// Log the error for monitoring
		log.Printf("Permission check failed: %v", err)
		
		// Fail closed: deny access when check fails
		return false
	}
	
	return response.Allowed == v1beta2.Allowed_ALLOWED_TRUE
}
```
